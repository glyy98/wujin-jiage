# 五金分类设计方案

## 首页交互（已实现）

- **左侧**：一级大类树状列表（「全部」+ 云数据库 `categories` 中 `parentId` 为空的项），点击切换选中态。
- **右侧**：当前选中大类下的商品明细；顶部保留在当前分类下的关键词搜索。
- 新增商品时在「新增商品」页选择「所属大类」，保存时写入 `categoryL1Id`、`categoryName`，便于首页按类筛选。
- 右侧已补充二级分类筛选条：默认首项「全部」，其后按 `order` 顺序展示；点击后按二级分类筛选商品。
- 首页点击「新增商品」会自动带入当前选中一级分类；若当前为「全部」，新增页默认首个一级分类。

**首次使用**：在云开发控制台调用云函数 `quickstartFunctions`，传入 `type: "initCategories"`，会初始化一批一级大类（紧固件、锁具、工具等）。若未初始化，左侧仅显示「全部」，右侧为全部商品。

## 一、现状

- 商品存于云数据库 `goods`，字段：name、spec、supplier、costPrice、salePrice、images
- 首页全量列表 + 关键词搜索（名称/规格/供应商）
- 类别多时：列表难找、新增时不知道选啥类、统计不方便

## 二、设计思路概览

| 方案要点 | 说明 |
|---------|------|
| **层级** | 建议 **二级分类**：一级大类（如紧固件、锁具、工具）→ 二级子类（如螺栓、螺母、垫片）。类别多时也够用，再需要可扩展为多级。 |
| **存储** | 单独 **分类集合** `categories`，商品表增加 `categoryId` 关联，便于维护和按类筛选。 |
| **维护** | 分类可做成 **可配置**：云数据库维护，后续加「分类管理」页；或先用一份内置常用分类快速上线。 |
| **交互** | 首页 **先选类再查** + 保留关键词搜索；新增/编辑商品时 **必选分类**，用级联或搜索选类。 |

## 三、数据模型

### 1. 分类集合 `categories`

```json
{
  "_id": "cat_xxx",
  "name": "螺栓",
  "parentId": "cat_fastener",   // 空或 null 表示一级分类
  "order": 1                    // 同层级排序，可选
}
```

- **一级**：`parentId` 为空（或 `""`）
- **二级**：`parentId` 为对应一级的 `_id`
- 查询：先查一级列表，再按 `parentId` 查二级

### 2. 商品集合 `goods` 增加字段

```json
{
  "categoryId": "cat_xxx",   // 指向二级分类 _id（或一级，看业务）
  "categoryName": "螺栓",    // 冗余，列表展示用，避免每次都联表
  // ... 原有 name, spec, supplier, costPrice, salePrice, images
}
```

- `categoryId` 必填，便于按类筛选、统计
- `categoryName` 冗余可减少联表，分类改名时需批量更新（或仅用 id 查分类表）

## 四、交互与页面

### 1. 首页（查询页）

- **分类筛选**  
  - 顶部：一级分类 Tab（横向滚动），如「全部」「紧固件」「锁具」「工具」…  
  - 点「全部」：展示全部商品（或最近/常用）  
  - 点某一级：可展开该一级下的二级，选二级后只显示该分类商品  
- **搜索**  
  - 保留现有关键词搜索；建议 **在已选分类下搜索**（先选类再搜，结果更准）。
- **列表**  
  - 每条可显示小标签：`categoryName`，方便辨认。

### 2. 新增/编辑商品（addGoods）

- **分类选择**  
  - 方式一：两级下拉——先选一级，再选二级（二级根据一级过滤）。  
  - 方式二：带搜索的 picker——输入关键字过滤「一级 + 二级」名称，再选一条（适合类别特别多）。  
- 必选一项二级分类（或一级，统一即可）再保存。
- 已落地为两级筛选框：先选「所属大类」，再选「所属二级分类」；二级默认「全部（可不选）」并允许不填。
- 进入「管理分类」或「管理供应商」后返回新增页，页面会自动刷新最新分类/供应商数据。
- 新增大类交互已更新：在「管理分类」旁新增「新增大类」按钮，弹框输入后可「添加并选用」。
- 新增供应商交互已简化：供应商块仅保留下拉选择，不再展示手动输入框。
- 规格支持顺序调整：点击「移动」后输入目标位置，可将任意规格移动到任意序号。
- 增加「规格单独配图」开关：  
  - 关闭：显示「商品图片（所有规格都用这些图）」；  
  - 开启：隐藏底部商品图片区，在每条规格下可上传/预览/删除规格图；  
  - 开关切换仅影响展示逻辑，已上传规格图数据会保留。

### 3. 商品列表页（goodsList）

- 顶部增加分类筛选（与首页一致：一级 Tab，可选二级），列表按当前筛选条件查询。

### 4. 分类管理（可选，后续）

- 单独页面：展示一级/二级树形列表，支持增删改、排序（order），仅管理员或指定用户可进。
- 已补充二级分类入口：在一级与其二级列表之间增加「+ 新增二级分类」按钮，可直接归属到当前一级。
- 已支持一级/二级同级排序：点击「移动」后输入目标位置，按序号重排。

## 五、云开发与性能

- **分类列表**  
  - 分类数量有限，可在首页/添加页 onLoad 时一次性拉取 `categories` 并缓存（如放 `getApp().globalData`），不必每次请求。
- **商品查询**  
  - 按 `categoryId` 筛选：`db.collection('goods').where({ categoryId: 'xxx' })`  
  - 组合：`categoryId + 关键词`（在 name/spec 等上做 match），注意云开发权限与索引。
- **索引**  
  - 为 `goods.categoryId` 建索引，便于按类分页、统计。

## 六、实施步骤建议

1. **建分类表与初始数据**  
   - 创建 `categories` 集合，写入一批一级/二级（可参考行业标准或你店里的习惯）。
2. **商品表扩展**  
   - 给 `goods` 增加 `categoryId`（及可选 `categoryName`）；旧数据可先设为「未分类」或批量按名称规则归到一个默认类。
3. **首页**  
   - 拉取分类 → 一级 Tab → 选二级 → 列表按 `categoryId` 查，保留搜索。
4. **addGoods**  
   - 增加分类选择（级联或可搜索），保存时写入 `categoryId`（和可选 `categoryName`）。
5. **goodsList**  
   - 同样增加分类筛选逻辑。
6. **（可选）分类管理页**  
   - 后续再做，用于增删改分类、排序。

## 七、小结

- **二级分类 + 独立 categories 表 + goods.categoryId** 能支撑「类别特别多」的浏览与筛选。
- 首页「先选类再查」+ 搜索，新增商品必选分类，列表页同样支持按类筛选，后续可加分类管理页做长期维护。
