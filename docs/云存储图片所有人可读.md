# 云存储图片「别人看不到」问题说明

## 原因

微信云开发的**云存储**有安全规则：默认或当前配置很可能是 **「仅创建者可读」**。

- 用户 A 上传的商品图片会带上 A 的访问身份。
- 用户 B 打开商品列表/详情时，用同一个 `fileID` 去拉取图片，云存储会校验「谁在访问」。
- 规则不允许 B 读 A 的文件时，请求被拒绝，图片就显示为**灰色/空白**。

所以现象是：**自己上传的能看，别人上传的不能看**。

## 解决办法（在控制台改权限）

需要把云存储改成：**所有人可读，仅上传者可写**。

### 操作步骤

1. 打开 **微信开发者工具**，点击左侧 **「云开发」**，进入云开发控制台。
2. 在云开发控制台左侧选 **「存储」**（云存储）。
3. 找到 **「权限设置」** / **「安全规则」**（不同版本可能名称略有差异）。
4. 将规则改为下面这一份（或在你现有规则里把 `read` 改为 `true`）：

```json
{
  "read": true,
  "write": "resource.openid == auth.openid"
}
```

含义：

- **read: true**：任意用户（包括未登录）都可以读文件，所以所有人都能看到别人上传的商品图。
- **write: "resource.openid == auth.openid"**：只有文件创建者（openid 与 resource 一致）可以上传/覆盖/删除，避免别人改或删你的图。

5. 保存后等待约 **1～3 分钟** 再在小程序里刷新预览，其他人就能正常看到图片。

### 参考

- [云存储安全规则 - 微信开放文档](https://developers.weixin.qq.com/miniprogram/dev/wxcloud/guide/storage/security-rules.html)

---

总结：**每个人只能看到自己上传的图、别人加的商品图片是灰色**，是因为云存储当前是「仅创建者可读」。在云开发控制台把存储的 `read` 设为 `true` 即可让所有人看到所有商品图片。
